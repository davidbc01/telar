# Cómo contribuir a Telar

Gracias por tu interés en contribuir. Telar está en sus primeras fases, así que cualquier aportación tiene un impacto real en la dirección del proyecto.

---

## El tipo de contribución más valioso ahora mismo

**Probar la sintaxis.** Escribe código Telar para tu caso de uso real y cuéntanos dónde la sintaxis se siente forzada, incómoda, o confusa. No necesitas saber programar compiladores para esto - de hecho, es mejor si no lo sabes.

Abre un issue con la etiqueta `diseño-sintaxis` y describe:
- Que querías expresar
- Cómo lo escribiste en Telar
- Por qué se sintió raro (si se sintió raro)

---

## Reportar un bug

1. Comprueba que no existe ya un issue similar
2. Abre un issue nuevo con la plantilla "Bug"
3. Incluye el archivo `.telar` que causa el problema
4. Describe qué esperabas que pasara y qué pasó

---

## Proponer una mejora de sintaxis

1. Abre un issue con la etiqueta `diseño-sintaxis`
2. Explica el caso de uso concreto que quieres resolver
3. Propón cómo se escribiría en Telar
4. Compara con cómo se haría en JavaScript/React

Las propuestas sin caso de uso concreto son difíciles de evaluar.

---

## Contribuir código

El proyecto todavía no acepta pull requests de código externo - estamos en una fase demasiado temprana donde la arquitectura cambia con frecuencia.

Esto cambiará en la v0.2. Si qieres contribuir código, abre un issue primero para coordinar.

---

## Código de conducta

Este proyecto sigue el [Contributor Covenant](https://www.contributor-covenant.org/).
En resumen: trato respetuoso, críticas al código no a las personas.
